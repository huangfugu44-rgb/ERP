## 一、项目概述

### 1.1 项目背景

本项目旨在基于开源jshERP系统的架构和功能，设计并开发一套全新的企业资源规划（ERP）管理系统。采用Vue 3前端框架和Node.js后端技术栈，实现前后端完全分离的现代化架构。

### 1.2 设计目标

- **功能完整性**：覆盖进销存+财务的完整业务流程
- **用户体验**：现代简约的设计风格，流畅的交互体验
- **技术先进性**：采用Vue 3、Node.js等主流技术
- **可扩展性**：模块化设计，支持二次开发和功能扩展
- **数据闭环**：每个功能点实现完整的业务闭环

## 二、模块一：首页/仪表盘模块

### 2.1 功能概述

仪表盘是系统首页，提供企业运营的关键数据总览和快速入口，帮助用户快速了解企业运营状况。

### 2.2 功能点清单

| 功能点 | 功能描述 | 优先级 | 闭环要求 |
|--------|----------|--------|----------|
| 今日概览 | 显示今日采购额、销售额、收款额、付款额 | P0 | 数据实时更新，点击可跳转明细 |
| 库存预警 | 显示低于安全库存的商品列表 | P1 | 点击商品跳转库存明细 |
| 应收提醒 | 显示接近到期的应收账款 | P1 | 点击跳转应收明细页面 |
| 应付提醒 | 显示接近到期的应付账款 | P1 | 点击跳转应付明细页面 |
| 待办事项 | 显示待处理的任务列表 | P0 | 支持完成任务、查看详情 |
| 近期订单 | 显示最近的销售/采购订单 | P1 | 点击跳转订单详情 |
| 快捷入口 | 提供常用功能的快速链接 | P2 | 支持自定义配置 |

## 三、模块二：零售管理模块

### 3.1 功能概述

零售管理模块处理面向终端客户的零售业务，包括零售单管理、零售退货、收银找零、零售日结、零售报表等功能。

### 3.2 功能点清单

| 功能点 | 功能描述 | 优先级 | 闭环要求 |
|--------|----------|--------|----------|
| 零售单列表 | 展示所有零售单据 | P0 | 支持筛选、搜索、导出 |
| 新建零售单 | 创建新的零售单 | P0 | 自动计算金额、支持多商品 |
| 零售单详情 | 查看零售单完整信息 | P0 | 包含商品明细、支付信息 |
| 收银找零 | 处理收款和找零 | P0 | 支持多种支付方式 |
| 零售退货 | 处理零售退货业务 | P1 | 关联原零售单、减库存 |
| 小票打印 | 打印零售小票 | P1 | 支持多种打印模板 |
| 零售日结 | 日结统计和账务核对 | P1 | 生成日结报表 |
| 零售报表 | 零售数据统计和分析 | P2 | 支持多维度统计 |

## 十二、数据库设计

### 12.1 数据库命名规范

- 表名使用小写字母，单词间用下划线分隔
- 字段名使用小写字母，单词间用下划线分隔
- 主键统一命名为 `id`
- 外键命名格式：`{表名}_id`
- 创建时间字段：`created_at`
- 更新时间字段：`updated_at`
- 软删除字段：`deleted_at`

## 十三、接口规范

### 13.1 RESTful API规范

```
基础URL: /api/v1

请求格式: Content-Type: application/json
认证方式: Bearer Token (JWT)

响应格式:
{
  "code": 200,           // 状态码
  "message": "success",    // 消息
  "data": {},            // 数据
  "timestamp": 1715606400 // 时间戳
}

状态码规范:
- 200: 成功
- 400: 请求参数错误
- 401: 未登录或Token过期
- 403: 无权限
- 404: 资源不存在
- 500: 服务器内部错误
```

### 13.2 认证接口

```
POST /api/v1/auth/login
请求: { username, password }
响应: { token, userInfo }

POST /api/v1/auth/logout
响应: { success }

GET /api/v1/auth/userinfo
响应: { id, username, displayName, roles, permissions }
```

### 13.3 分页接口

```
GET /api/v1/{resource}
Query参数:
- page: 页码（默认1）
- pageSize: 每页条数（默认20）
- sort: 排序字段
- order: 排序方向（asc/desc）
- keyword: 关键词搜索

响应:
{
  "list": [],           // 数据列表
  "total": 100,         // 总条数
  "page": 1,            // 当前页
  "pageSize": 20,       // 每页条数
  "totalPages": 5      // 总页数
}
```
